### EP5-S3: Forbidden Terms

#### Backend

1. **ForbiddenCheckController** (`/api/forbidden-check`):
   - POST — body: `{ text, languageCode, workspaceId }` → return array of `{ term, replacement, position }`
   - Case-insensitive matching of text against all forbidden terms (where IsForbidden=true) in workspace glossaries
   - Return each match with the forbidden term, its replacement, and character position in text

2. **ContentItemLanguageTask update** — when saving a translation, if forbidden terms detected, set a `RequiresReview` flag. Add `RequiresReview` bool property to `ContentItemLanguageTask` entity if not present.

3. Add EF migration if entity changes needed.

### EP5-S2: Style Rules Engine

#### Backend — New Entities (add to Entities.cs)

```csharp
public sealed class StyleRule
{
    public Guid Id { get; set; } = Guid.NewGuid();
    public Guid ProjectId { get; set; }
    public required string Name { get; set; }
    public required string RuleType { get; set; }  // "case_check", "regex", "no_trailing_punctuation", "max_words"
    public string Pattern { get; set; } = string.Empty;  // regex for "regex" type
    public string Scope { get; set; } = string.Empty;  // content type filter or empty for all
    public string Message { get; set; } = string.Empty;  // warning message
    public bool IsActive { get; set; } = true;
    public DateTime CreatedUtc { get; set; } = DateTime.UtcNow;
}

public sealed class StyleOverride
{
    public Guid Id { get; set; } = Guid.NewGuid();
    public Guid ContentItemLanguageTaskId { get; set; }
    public Guid StyleRuleId { get; set; }
    public required string OverriddenByEmail { get; set; }
    public DateTime CreatedUtc { get; set; } = DateTime.UtcNow;
}
```

#### Backend — AppDbContext
- Add DbSets for StyleRule and StyleOverride
- StyleRule: Name max 200, RuleType max 50, Pattern max 1000, Scope max 50, Message max 500, index on ProjectId
- StyleOverride: unique index on (ContentItemLanguageTaskId, StyleRuleId), FK to both

#### Backend — Controllers

**StyleRulesController** (`/api/projects/{projectId}/style-rules`):
- GET — list rules for project
- POST — create rule
- PUT `/{ruleId}` — update
- DELETE `/{ruleId}` — delete

**StyleCheckController** (`/api/style-check`):
- POST — body: `{ text, projectId, contentType }` → return violations array: `[{ ruleId, ruleName, message, ruleType }]`
- Rule evaluation logic:
  - `case_check`: if text has Title Case words, warn for sentence case
  - `regex`: test text against Pattern, if matches → violation
  - `no_trailing_punctuation`: check if text ends with . or !
  - `max_words`: count words, if exceeds limit (stored in Pattern as number) → violation
- Only evaluate rules where Scope matches contentType (or Scope is empty = applies to all)

#### Backend — Migration
Run: `dotnet ef migrations add AddStyleRuleEntities --project ContentLocalizationSaaS.Infrastructure --startup-project ContentLocalizationSaaS.Api`
